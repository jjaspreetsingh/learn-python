Classes & Objects

Object-Oriented Programming (OOP) allows you to model real-world things in your code. A Class is a blueprint; an Object is a house built from that blueprint.

1. The Blueprint (Class)

We define a class using the class keyword. Inside, we define functions (methods) and data (attributes).

class Dog:
    def __init__(self, name, breed):
        self.name = name
        self.breed = breed
    
    def bark(self):
        return "Woof! I am " + self.name

Concept: __init__ is a special method (constructor) that runs automatically when you create a new object. self refers to the specific object being created.

2. Creating Objects

Now we can create "instances" of our Dog class.

# Create two dogs
dog1 = Dog("Buddy", "Golden Retriever")
dog2 = Dog("Rex", "German Shepherd")

print(dog1.bark()) # Output: Woof! I am Buddy
print(dog2.breed)  # Output: German Shepherd

3. Why use Classes?

Classes help organize complex code. Instead of having loose variables like dog1_name, dog1_breed, dog2_name, etc., you group related data and behavior together.
