Project: Automated Deployment

Don't do manual work. Write a Python script to test, build, and "deploy" your application automatically.

1. The Goal

We want a script that runs commands for us. It should:

  1. Run tests (`pytest`).
  2. If tests pass, create a backup.
  3. "Deploy" the new code (copy files).

2. The Automation Code

import subprocess
import shutil
import datetime
import os

def run_step(command, step_name):
    print(f"--- Starting {step_name} ---")
    result = subprocess.run(command, shell=True)
    if result.returncode == 0:
        print(f"‚úÖ {step_name} Success!")
        return True
    else:
        print(f"‚ùå {step_name} Failed.")
        return False

def deploy():
    # 1. Run Tests
    if not run_step("echo 'Running tests...' && sleep 1", "Tests"):
        return

    # 2. Backup
    timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
    backup_dir = f"backup_{timestamp}"
    print(f"Creating backup: {backup_dir}")
    os.makedirs(backup_dir, exist_ok=True)
    # Simulate backup
    with open(f"{backup_dir}/backup_log.txt", "w") as f:
        f.write("Backup complete.")
    
    # 3. Deploy
    print("Deploying to production server...")
    # In real life, this might use 'scp' or 'rsync'
    print("üöÄ Deployment Complete!")

if __name__ == "__main__":
    deploy()

3. Why Python for DevOps?

Bash scripts are great, but Python gives you better logic, error handling, and libraries. It's the language of choice for cloud automation.
